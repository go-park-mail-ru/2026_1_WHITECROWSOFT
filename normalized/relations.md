# Нормализаванная схема данных

## 1. Таблица `note` (Заметки)
Информация о заметках, которые создают пользователи. Поддерживает древовидную структуру.
#### Функциональные зависимости:
{id} -> {user_id, title, parent_id, created_at, updated_at}
#### Нормализации:
-   **1НФ:** Атомарные значения. Одному атрибуту - одно значение.
-   **2НФ:** Первичный ключ — `id`. Неключевые атрибуты зависят от всего первичного ключа.
-   **3НФ:** Неключевые атрибуты (`title`, `user_id`) зависят только от первичного ключа `id`. `parent_id` ссылается на другой `id` в этой же таблице, что является рекурсивной зависимостью, но не транзитивной зависимостью, т.к. `parent_id` не определяет другие атрибуты (`title` не зависит от `parent_id`).
-   **НФБК:** Единственный детерминант — ключ `id`.

## 2. Таблица `account` (Пользователи)
Информация о зарегистрированных пользователях.
#### Функциональные зависимости:
{id} -> {username, password, token_version, created_at, updated_at}
{username} -> {id, password, token_version, created_at, updated_at} (потенциальный ключ)
#### Нормализации:
-   **1НФ:** Атомарные значения. Одному атрибуту - одно значение.
-   **2НФ:** Первичный ключ — `id`. Неключевые атрибуты (`username`, `password`, `token_version`) зависят от всего первичного ключа.
-   **3НФ:** Все неключевые атрибуты (`username`, `password`, `token_version`) зависят только от первичного ключа `id`.
-   **НФБК:** Единственные детерминанты — это потенциальные ключи (`id` и `username`).

## 3. Таблица `block` (Блоки)
Заметка состоит из блоков различных типов (например, текст, изображение и т. п.), упорядоченных по полю `position`.
#### Функциональные зависимости:
{id} -> {note_id, position, block_type_id, content, created_at, updated_at}
#### Нормализации:
-   **1НФ:** Атомарные значения. Одному атрибуту - одно значение.
-   **2НФ:** Первичный ключ `id`, неключевые атрибуты зависят от всего первиченого ключа.
-   **3НФ:** Неключевые атрибуты (`note_id`, `position`, `block_type_id`, `content`) зависят только от конкретного `id`.
-   **НФБК:** Единственный детерминант — ключ `id`.

## 4. Таблица `block_type` (Типы блоков)
Список возможных типов блоков.
#### Функциональные зависимости:
{id} -> {name}
{name} -> {id}
#### Нормализации:
-   **1НФ:** Атомарность.
-   **2НФ:** Первичный ключ `id`, неключевые атрибуты зависят от всего первиченого ключа.
-   **3НФ:** Неключевой атрибут `name` зависит от первичного ключа `id`.
-   **НФБК:** Детерминанты — `id` и `name` (потенциальный ключ).

## 5. Таблица `block_state` (Состояние блока)
Форматирование для блока.
#### Функциональные зависимости:
{id} -> {block_id, formatting, created_at, updated_at}
#### Нормализации:
-   **1НФ:** Атомарность. Предполагается, что `formatting` — это текст в некотором формате (строка).
-   **2НФ:** Первичный ключ `id`, неключевые атрибуты зависят от всего первиченого ключа.
-   **3НФ:** Все атрибуты зависят только от `id`.
-   **НФБК:** Единственный детерминант — ключ `id`.

## Описание таблиц:

### `note` - заметка
- id - уникальный идентификатор заметки.
- user_id - идентификатор владельца заметки.
- title	- заголовок заметки.
- parent_id	- идентификатор родительской заметки, позволяет создавать вложенные заметки. Если поле NULL, заметка находится в корневом каталоге.
- created_at - дата и время создания заметки.
- updated_at - дата и время последнего обновления заметки.

### `account` - пользователи

- id - уникальный идентификатор пользователя.
- username - логин пользователя, используемый для входа.
- password - хэш пароля пользователя. 
- token_version - версия jwt-токена.
- created_at - дата и время создания аккаунта.
- updated_at - дата и время последнего обновления данных 

### `block` - блоки данных

- id - уникальный идентификатор блока.
- note_id - идентификатор заметки, которой принадлежит данный блок.
- block_type_id	- идентификатор типа блока.
- position - номер блока внутри заметки.
- content - хранящаяся в блоке информация.
- created_at - дата и время создания блока.
- updated_at - дата и время последнего обновления блока.

### `block_type` - тип блока

- id - уникальный идентификатор типа блока.
- name - название типа блока.

### `block_state` - cостояние блока

- id - уникальный идентификатор записи состояния.
- block_id - идентификатор блока, к которому относится данное состояние.
- formatting - данные состояния блока (строка).
- created_at - дата и время создания данной версии состояния.
- updated_at - дата и время последнего обновления этой записи состояния.
